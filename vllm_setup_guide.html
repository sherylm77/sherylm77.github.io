Setting up vLLM on CARC

This guide explains how to install and set up vLLM on CARC to use in your experiment scripts. Using vLLM is great because you can easily switch between different models without changing the way you prompt them.

What vLLM is doing under the hood

vLLM runs as a model inference server. When you launch it, it loads a model into GPU memory and exposes it through an HTTP API endpoint (similar to OpenAI's API).

This means:

  • You interact with the model by sending requests to a base URL (e.g., http://<node>:<port>/v1)
  • Your Python scripts don't talk to the model directly-they talk to the vLLM server
  • The server handles batching, GPU execution, and response generation

Because vLLM uses an OpenAI-compatible API, you can reuse the same client code across different models and switch models by changing the model field or endpoint without rewriting prompts.

Installation

See vLLM docs

  1. SSH into CARC
  2. Load CUDA: 
    module load cuda
  3. Double check that CUDA_HOME is in PATH: 
    echo $CUDA_HOME
echo $PATH
  4. Install vLLM: 
    pip install vllm

Setup

1. Start a session

Open a new tmux window

2. Request GPUs on CARC

srun --pty --partition=nlp_hiprio --gres=gpu:rtxa6000:n --cpus-per-task=10 \
--mem=100GB --account=xiangren_1715 -t 1-10 bash

Replace n with number of GPUs. Note the compute node (e.g. c05-04).

3. Launch the vLLM instance

a. Create a deploy script

  • Replace model_name with the HuggingFace model name
  • n is the number of GPUs
  • api_key can be a placeholder like "test"
VLLM_ALLOW_LONG_MAX_MODEL_LEN=1 VLLM_USE_V1=1 vllm serve model_name \
            --port 31000 \
            --dtype auto \
            --tensor-parallel-size n \
            --host 0.0.0.0 \
            --api-key api_key

b. Run the server

bash deploy.sh
  • This may take time initially (downloads + model loading)
  • You should see this message when the model has loaded: INFO: Application startup complete.

4. Run your Python script

from openai import OpenAI

api_key = "test"
base_url = "http://<compute_node>.hpc.usc.edu:<port>/v1"

client = OpenAI(
    api_key=api_key,
    base_url=base_url,
)

response = client.chat.completions.create(
    model=<model_name>,
    messages=[
        {"role": "system", "content": "You are a helpful assistant."},
        {"role": "user", "content": "Hello!"},
    ],
)

print(response.choices[0].message.content)

Important notes

  • Use the same API key as in deployment
  • The base URL is made from compute node + port
http://c05-02.hpc.usc.edu:31000/v1
  • model_name must match the deployed model

Using OpenAI models instead

  1. Change:
    • api_key -> your OpenAI API key
    • base_url -> https://api.openai.com/v1
    • model_name -> e.g. gpt-5-mini
